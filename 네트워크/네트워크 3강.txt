교환시스템
	교환 시스템 : 네트워크 양단에 연결된 호스트들이 전송하는 패킷은 전송 경로 중간에 위치한 교환 시스템을 거침(데이터를 최종 목적지 까지 올바른 경로로 중개하는 교환 기능제공)
	연결형서비스) 음성전화서비스, 고정대역폭, 구조가 단순
	-> 회선교환방식 은 고정대역으로 할당된 연결을 설정, 데이터전송시작(안정적).(연결이 유지되는 동안 다른연결에서 이 대역 사용불가 --- 라우터)
	비연결형서비스) 컴퓨터네트웤, 가변대역폭, 구조가 복잡함.   둘다 패킷교환과정 이야기
	-> 패킷 교환방식 - 네트워크환경이용 , 데이터를 미리 패킷으로 나누어 전송한다.(패킷기준 교환작업)
	또 전용 대역이 없으니 가변 대역크기의 전송률로 이루어진다. 
	모든패킷의 경로를 일정하게 유지시키는 데이터그램, 가상회선 방식있다.
	
	교환시스템 종류
	1) 전용회선방식 : 송수신호스트 전용 할당된 통신선로로 데이터 전송방식. -> 안씀
	//2) 교환회선방식 : 전송선로하나로 다수의 사용자 공유 연결을 미리 하지 않고 데이터를 패킷 단위로 나누어 전송하는 패킷 교환 방식임.
	
	1.1) 회선 교환
	호스트가 데이터를 전송하기 전에 연결 경로를 미리 설정하는 방식임. 연결과정에서 모든 경로 결정 -> 모두 같은 경로 -> 고정 대역의 논리적인 전송 선로, 안정적인 데이터 전송률
	
	1.2) 메시지 교환
	데이터를 전송하기 전에 경로를 미리 설정 안하고 대신 전송하는 메시지의  헤더마다 목적지 주소 표기하는 방식.
	송수신 호스트가 보내고 받은 시점이 반드시 일치할 필요없음 ㅇㅇ
	
	1.3) 패킷 교환
	송신호스트는 전송할 데이터를 패킷이라는 일정 크기로 나누어 전송한다. 각 패킷은 독립적으로 라우팅 과정을 거쳐 목적지에 도착 함
	장점 : 
	전송 대역의 효율적 이용 : 여러 호스트에서 전송한 패킷들이 동적인 방식으로 공유하기에... 장점이 된다.
	무제한수용 : 임의의 연결요청에 고정 대역을 할당 안함 무한이긴한데.. (호스트가 많아지면 혼잡도 높아짐.. 전송지연이랑)vs 회선교환방식에서는 개별 연결 요청에 대해 고정대역할당. 초과불과.
	패킷의 우선순위 부여 : 우선순위를 부여할 수있다 큰장점임 !!
	
	단점  :  회선교환방식보다 지연율이 높다.
	-- 패킷경로 2개 정적경로(고정)/동적경로

	- 가상 회선 방식-
	데이터를 패킷 단위로 나누어 전송하거나 송수신 호스트 사이에 가상 연결을 설정하므로 모든 패킷의 전달 경로가 동일함.
	연결형 서비스 지원이다.
	-데이터 그램-
	패킷의 경로 선택이 독립적 + 가변적
	비연결형 서비스 지원임. 정보의 양이 적거나, 신뢰성이 중요하지 않는 경우.

	Lan, Man Wan
	네트워크구성방식
	Lan은 소규모 네트워크임. 주로 많이 사용하는 네트웤 연결형태는 링인데 버스를 소개할게. 
	버스는 공유버스하나에 여러 호스트를 직접 연결해서 쓰지 한 호스트가 전송한 데이터를 모든 네트웤에 연결된 호스트에 전송하므로 브로드 캐스팅 방식이야.
	또한 둘이상의 호스트에서 공유하려고하면 충돌현상도 일어나.
	링형은 충돌방지나 파티션(분리)에서 매우 좋아 어떤 인터페이스가 없어졌을 때 그 곳을 통하지 않더라도 딴곳을 통해 연결이 가능하지.
	주로 매쉬형태로 안전성을 높이고 트래픽스라고 ipv4 3개의 주소번지를 이용 + 라우터 4개이상 서로연결.
	링형은 토큰이라는 제어프레임을 사용해 충돌가능성 방지.

	Man은 도시단위 네트웤.

	3) 인터넷 라우팅)
	경로지정이 매우중요해...
	1) 고정 경로 배정
	네트워크의 구성이 변경이 다 끝난 후 가능.
	간단 구현방식. 영구불변의 경로 배정
	단점은 트래픽 변화에 따른 동적 경로 배정이 불가하다.
	경로배정은 데이터전송량에 의해 측정됨. 

	2) 적응 경로 배정
	인터넷에서 사용되는 라우터 경로 배정 방식임.
	혼잡이나 다른 라우터가 제대로 동작안하면 경로에 반영!
	단점 : 경로복잡 -> 라우터 부담 -> 동시반영 불가능 -> 해결책으론 정보교환을 빠르게.... -> 그러나 트래픽증가 -> 처리부담 ㅠㅠ

	3) 자율시스템
	다수의 라우터 구성-> 공통의 라우터 프로토콜 사용 -> 논리적 단일 구성체임
	그리고 자율시스템간에 사용하는 라우팅 프로토콜을 일반적으로 외부 라이팅 프로토콜이라함.(내부 RIP, OSPF 외부 BGP)

	
	서비스 품질(QoS) : 데이터를 어느정도로 신뢰성 있게 전달하는가...
